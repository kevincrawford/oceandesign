Grid and Recordview Use Cases

When grid is used to navigate to a record(s) that require further actions

When all data fields don't fit in a grid without horizontal scrolling or reading the data becomes difficult

When the grid column model represents a subset of the entire record model and details need to be exposed to support the desision process

When records are selected to take an action on

Grid and Recordview alway present

User wants to navigate to and view multiple record details*

  1. When UI can accomodate grid and record view
  2. When required visible grid columns < 3

*Requires small screen solution

Grid and Recordview Popover

User needs to quickly see additional record details to support action decisions

  1. When grid is primary navigation or scanning for records
  2. When Recordview information can fit in a small container (240px X 240px or popover does not obscure more tha 25% of grid)
  3. When required visible grid columns > 5
  4. When multiple records will be selected

Grid and Recordview Flyout

User needs to focus on record details in a readable format

Recordview is hidden until it is needed. When recordview is needed it opens into the grid leaving 1-3 columns available in view to support navigation

  1. When recordview is too large for popover
  2. When user needs navigate to and view multiple records details
  3. When multiple records will be selected

Grid and Row Expansion

User needs to quickly see additional record details to support action decisions

Last resort option

  1. Only use when the format of data supports the existing grid column layout (eg. Additional information matches grid column model, or information provides additional information for a span of columns
